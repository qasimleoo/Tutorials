## 1. Conceptual Difference (High Level)

### Spring Framework

* Core framework hai
* Dependency Injection (DI) provide karta hai
* Aap ko **sab kuch manually configure** karna hota hai
* Web support built-in nahi hota, MVC alag se setup karna padta hai

Use case: Jab lightweight core features chahiyein

---

### Spring MVC

* Spring Framework ka **web module**
* Web applications aur REST APIs ke liye
* DispatcherServlet, Controllers, ViewResolvers sab manually configure
* XML ya Java config zyada hoti hai

Use case: Traditional web applications

---

### Spring Boot

* Spring Framework + Spring MVC ka **auto-configured version**
* Embedded server (Tomcat)
* XML almost zero
* Production-ready, fastest development

Use case: Microservices, REST APIs, modern apps

## A. Spring Framework (Core only)

### Configuration Class

```java
import org.springframework.context.annotation.Bean;
import org.springframework.context.annotation.Configuration;

@Configuration
public class AppConfig {
    @Bean
    public HelloService helloService() {
        return new HelloService();
    }
}
```

### Service Class

```java
public class HelloService {
    public String message() {
        return "Hello Spring";
    }
}
```

### Main Class

```java
import org.springframework.context.ApplicationContext;
import org.springframework.context.annotation.AnnotationConfigApplicationContext;

public class MainApp {
    public static void main(String[] args) {
        ApplicationContext context =
                new AnnotationConfigApplicationContext(AppConfig.class);
        HelloService service = context.getBean(HelloService.class);
        System.out.println(service.message());
    }
}
```

Output console pe print hota hai
Web app nahi hai

## B. Spring MVC

### Controller

```java
import org.springframework.web.bind.annotation.GetMapping;
import org.springframework.web.bind.annotation.RestController;

@RestController
public class HelloController {
    @GetMapping("/hello")
    public String hello() {
        return "Hello Spring";
    }
}
```

### Web Initializer

```java
import org.springframework.web.servlet.support.AbstractAnnotationConfigDispatcherServletInitializer;

public class WebInit extends AbstractAnnotationConfigDispatcherServletInitializer {
    protected Class<?>[] getRootConfigClasses() {
        return null;
    }
    protected Class<?>[] getServletConfigClasses() {
        return new Class[]{WebConfig.class};
    }
    protected String[] getServletMappings() {
        return new String[]{"/"};
    }
}
```

### Config

```java
import org.springframework.context.annotation.ComponentScan;
import org.springframework.context.annotation.Configuration;
import org.springframework.web.servlet.config.annotation.EnableWebMvc;

@Configuration
@EnableWebMvc
@ComponentScan
public class WebConfig {
}
```

Run on external Tomcat
URL:

```
http://localhost:8080/hello
```

## C. Spring Boot

### Application Class

```java
import org.springframework.boot.SpringApplication;
import org.springframework.boot.autoconfigure.SpringBootApplication;

@SpringBootApplication
public class DemoApplication {
    public static void main(String[] args) {
        SpringApplication.run(DemoApplication.class, args);
    }
}
```

### Controller

```java
import org.springframework.web.bind.annotation.GetMapping;
import org.springframework.web.bind.annotation.RestController;

@RestController
public class HelloController {
    @GetMapping("/hello")
    public String hello() {
        return "Hello Spring";
    }
}
```

Run command:

```
mvn spring-boot:run
```

URL:

```
http://localhost:8080/hello
```

No XML
No external server
Minimum code

## 3. Side-by-Side Comparison

| Feature        | Spring   | Spring MVC | Spring Boot |
| -------------- | -------- | ---------- | ----------- |
| Web support    | No       | Yes        | Yes         |
| Configuration  | Manual   | Heavy      | Auto        |
| Server         | N/A      | External   | Embedded    |
| XML            | Possible | Mostly     | Rare        |
| Learning curve | Medium   | High       | Low         |
| Speed          | Slow     | Medium     | Fast        |

Entry point
endpoints/apis/controllers
1. Controller (@RestController - @Controller)
get the request .. process it .. and return the response (no business operations here)

2. Service
controller forwards the request to service (business operations happen in service layer - all data operations .. populations .. processing)

3. Repository
Service layer fetches data from repositoy layer (that connects to DB to get the requested/resepctive data)

Servive layer can directly get the data from DB .. but that's not ideal and breaks the layers functionality


Now let's talk about above four things

i. DTO (data transfer object)
let's say client can request a post request with a request body (json, toon).. so that body/data should be converted to some java object inside controller which we call as DTO 

ii. Entity
a mapping to db (details of your db tables or any other things inside data)

for a response Entity is converted to DTO inside service layer

iii. Utility
Something common between all the layers 

iv. Config (third party or something that is insdie properties files)
@Value

10 mistakes that java dev make

i.
foreach loop
you can't break it
can't remvoe elements while looping it

to remove you have to use iterator

```
ArrayList<String> words = new ArrayList<>(
    Array.asList("1","2","3","4")
)

// INLVAID
words.forEach(word -> {
    if (word.equals("1")) {
        words.remove(word);
    }
})

// VALID
Iterator<String> iterator = words.iterator();
while (iterator.hasNext()) {
    String word = iterator.next();
    if (word.equals("1")) {
        iterator.remove();
    }
}

// or we can replace while loop here with Collections.removeIf()
words.removeIf(word -> word.equals("1"));
```

ix. 

excessive_garbage_collection

```

for (int i = 0; i < 1_000_000, i++) {
    out += i;
}
System.out.print(out);
```

instead use string builder
as strings are immutable .. so under the hood it makes a new string builder copies content discards new string .. appends new and makes it O(n^2)


- Few concatenations, known at compile time → String
- Loops / large / unknown size → StringBuilder
- Multithreaded shared builder → StringBuffer (rare)

```
Character func() {
    String name = null;
    if (name == null) {
        return null;
    }
    return name.charAt(0);   
}

// replace with
Optional<Character> func() {
    String name = null;    
    if (name == null) {
        return Optional.empty();
    }
    return Optional.of(name.charAt(0));
}
```
